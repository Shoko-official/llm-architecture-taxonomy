# Roadmap

This roadmap starts with governance only. Taxonomy content starts after
Milestone 0 is complete and the repository has validation and CI.

## Milestone 0: Project Governance

Goal: establish a small, reviewable operating model for the architecture
taxonomy repository.

### Issues

1. [#1 Create taxonomy governance documentation](https://github.com/Shoko-official/llm-architecture-taxonomy/issues/1)
2. [#2 Add issue and PR templates](https://github.com/Shoko-official/llm-architecture-taxonomy/issues/2)
3. [#3 Add minimal validation, CI, and folder structure](https://github.com/Shoko-official/llm-architecture-taxonomy/issues/3)

### Execution Order

1. Complete issue #1 before templates or validation.
2. Complete issue #2 before content changes.
3. Complete issue #3 before taxonomy content.

No taxonomy content should be added during Milestone 0.

## Acceptance Criteria

Milestone 0 is complete when:

* repository role is documented;
* roadmap exists;
* contribution and review rules exist;
* issue and PR templates exist;
* minimal validation commands exist;
* minimal CI exists;
* initial folders exist;
* no taxonomy content has been added.

## Later Milestones

Expected sequence:

1. Define a minimal taxonomy skeleton.
2. Define architecture layer boundaries.
3. Add neutral architecture term stubs.
4. Add evidence handoff rules from the research ledger.
5. Add validation for taxonomy files.
6. Introduce diagrams only through Mermaid or approved Python-generated figures.

Any change to taxonomy structure, diagrams, or evidence rules must happen in a
dedicated issue.
